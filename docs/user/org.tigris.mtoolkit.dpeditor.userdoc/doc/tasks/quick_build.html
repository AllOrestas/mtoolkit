Quick Deployment Package Build

The Deployment Package Editor allows the quick generation of a deployment package after its properties have been preliminary defined in the Deployment Package Project file. Use the Build tab of the editor for the corresponding Deployment Package Project file to change the default file path for the deployment package generation.

To use the internal generator of the Deployment Package Editor, select the Deployment Package Project file (.dpp) for the targeted deployment package in one of the navigation views, right-click on the node and use the Quick Build option from the pop-up menu.

Note: In case some bundle resources participating in the generated deployment package have different JAR file names from the saved in the Deployment Package Project file, depending on the configuration settings you have made for the Deployment Package Editor you will or will not be informed of the bundle file names currently used.

Note: If you will sign the generated deployment package but you have not entered the password(s) related to the signer(s), a dialog is displayed requesting the missing password(s) to be specified. Otherwise, the signer information specified in the Deployment Package Project file is not taken into consideration for the current deployment package generation.
