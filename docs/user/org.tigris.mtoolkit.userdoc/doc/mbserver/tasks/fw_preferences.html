Customizing OSGi Management

mToolkit allows you to customize remote management of OSGi frameworks - you can configure mToolkit to automatically connect to frameworks which have been already connected by third party Eclipse plug-ins, or to have information about OSGi management logged:

  1. Select Window > Preferences .
  2. Browse the Preferences dialog and locate the mToolkit preference page.
  3. Under the Remote Management preference category if needed use the following options:
    • Activate automatic connection to OSGi framework instances when a third party plug-in opens such a connection - check the Enable OSGi Frameworks automatic connection option.
    • Activate logging of events in the Error Log view - check the Enable extended logging option.
    • To have a bundle automatically started upon install, select Autostart bundles on install.
    • To view the bundles of a connected framework in the Frameworks view grouped by category, select Group bundles in categories.
  4. Under Certificates specify the own certificates (certificate and private/public key pair) which can be used to sign bundles and deployment packages upon deployment on a connected OSGi framework. Signing is activated individually for each OSGi framework in the relevant connection settings (see "Adding/Removing an OSGi Framework").

    Use the Add button to specify a signer by keystore type and location, and signer alias within the keystore.

    Note: To be able to use the alias for signing, the passwords protecting the keystore and the private key associated with the alias must be the same.

  5. To save the changes:
    • To apply the changes and exit the Preferences dialog, click OK.
    • To apply the changes and to continue using the Preferences dialog for setting specific preferences, click Apply.
