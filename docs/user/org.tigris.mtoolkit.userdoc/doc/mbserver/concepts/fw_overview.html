OSGi Management Overview

The OSGi frameworks management capabilities of mToolkit allows managing bundles and optionally deployment packages on remote OSGi frameworks directly from the development environment. This feature is useful for testing behavior of deployable units (bundles, deployment packages, applications, etc) in a real OSGi framework environment, and bug-fixing, profiling and optimizing at runtime. You can change the content of the deployable unit after having installed it on the framework, and update it right away.

Management Model

mToolkit allows you to create a list of OSGi frameworks and connect to any of them by simply selecting it from the list and invoking "connect". The list of OSGi frameworks is placed in a separate view - the Frameworks view.

Once connected, the framework can be ordered to install, start, stop, update and uninstall specific bundles, or install and uninstall deployment packages, MIDlets and Opera Widgets. You can also manage the applications compatible with the OSGi Application Admin specification currently installed on the framework.

Communication with OSGi Frameworks

The communication with an OSGi framework is acquired over the Instrumentation Agent (IAgent) running on top of it.

Setting up the OSGi Framework

To be able to connect your OSGi framework to mToolkit and to Eclipse, you have to install the IAgent OSGi bundle in it. The IAgent OSGi bundle is available here:

<eclipse_home_dir>/dropins/mToolkit/iagent/org.tigris.mtoolkit.iagent.rpc_Xxx.jar

Otherwise, in the Frameworks view try to connect to the OSGi framework with missing IAgent and use the Get IAgent option from the displayed error dialog to save the bundle on yout computer.

Basically, the IAgent OSGi bundle needs the OSGi Package Admin and Start Level services and the Service Tracker. To use its full capabilities, the OSGi framework should be running its console parser, the OSGi Event Admin service and the Deployment Admin service. For the needs of application management, the framework should also have the OSGi Application Admin service along with a proper application container.
