There are more then one way to install additional plug-ins in Eclipse, which are listed below. Note that it is not advisable to use both or switch from to another. Eclipse p2 is not mature enough (as of January 2010) to handle this and will often result in broken Eclipse configuration, which are almost impossible to fix.

Eclipse Update Site/p2 Repository

This is the preferred method for installing new plug-ins. More information about can be found in Eclipse documentation. You'll need to grab the link to Eclipse Update site above.

dropins folder

This is less supported method for installing plugins. You just grab the build archive and extract it in dropins/ folder, found in the Eclipse installation. For better organization, it is advisable to first create new folder, which will hold mToolkit installation. Your tree structure should look like this:

<eclipse>/
 dropins/
  mToolkit/
   eclipse/
    features/
    plugins/

All plug-ins from the archive should end up in the plugins/ folder shown above.
<eclipse> is your Eclipse installation folder.
