What is Remote OSGi Management

Remote OSGi Management plug-in gives you ability to inspect and manage a remotely running OSGi framework. The view gives you quick access to the list of installed bundles, their headers, registered and used services and ability to manage bundles' lifecycle - install/uninstall/start/stop is all there, at the tip of your fingers. Additionally, Remote OSGi Management plug-in supports Deployment Admin Specification (Service Compendium, R4.1, ch. 114) and Application Admin Service Specification (Service Compendium, R4.1, ch. 116). This means that you are able to install/uninstall/inspect deployment packages and list/start/stop applications, which conforms to the Application Admin Service Specification directly from Eclipse.

How to connect to a remote OSGi framework

You can connect to any R4-compliant OSGi framework after you had installed a small bundle, which gives access to the internals of the remote OSGi framework.

Although the bundle is distributed with mToolkit1, it is easier to get it directly from http://mtoolkit.tigris.org/rel/iagent.rpc.jar. You can either deliver the downloaded bundle to the device and install it locally or directly install it from online source (if your device have Internet connectivity). After the bundle is installed, it must be activated (transition to ACTIVE state). For more information on how to install and activate the bundle, please consult the documentation of the your OSGi framework. After the bundle is installed and active, you are ready to connect to the remote OSGi framework.

  1. Open Frameworks view by selecting Window -> Show View -> Other... and select mToolkit -> Frameworks entry.
  2. Add new framework, selecting the Add Framework button and entering the name and the IP of the remote framework.
  3. Close the dialog by selecting Connect button. This will connect to the remote framework immediately. The list of installed bundles should be retrieved shortly.
You can start exploring the view.
