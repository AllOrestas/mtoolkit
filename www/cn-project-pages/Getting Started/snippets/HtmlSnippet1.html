What is Remote OSGi Management

Remote OSGi Management plug-in gives you ability to inspect and manage a remotely running OSGi framework. The view gives you quick access to the list of installed bundles, their headers, registered and used services and ability to manage bundles' lifecycle - install/uninstall/start/stop is all there, at the tip of your fingers.

How to connect to a remote OSGi framework

You can connect to any R4-compliant OSGi framework after you had installed a small bundle, which gives access to the internals of the remote OSGi framework.

  1. Navigate to /plugins/ folder and locate org.tigris.mtoolkit.iagent.rpc_2.0.0.xxxx.jar plugin. Note: is your Eclipse installation folder, where mToolkit is installed.
  2. Install the selected bundle in OSGi framework and activate it. For more information on how to install and activate a bundle, please consult the documentation of your OSGi framework vendor.
  3. Confirm that the bundle is properly installed and in ACTIVE state.

After the above steps, you are ready to connect to the remote OSGi framework.

  1. Open Frameworks view by selecting Window -> Show View -> Other... and select mToolkit -> Frameworks entry.
  2. Add new framework, selecting the Add Framework button and entering the name and the IP of the remote framework.
  3. Close the dialog by selecting Connect button. This will connect to the remote framework immediately. The list of installed bundles should be retrieved shortly.
You can start exploring the view.
